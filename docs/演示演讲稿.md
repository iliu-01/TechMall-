## 开场（30 秒）

各位老师好，我是 [姓名]，我展示的项目是 **TechMall 数码商城**——一个基于 Spring Cloud Alibaba 微服务架构的 3C 数码产品垂直电商平台。

项目完整实现了用户、商家、管理员三种角色的全业务流程，涵盖从注册登录、商品浏览、下单支付，到商家发货、用户确认收货的完整闭环。技术栈严格按照考核要求：JDK 17 + Spring Boot 3.3.4 + Spring Cloud Alibaba 2023.0.3.2 + Nacos + Sentinel + Gateway + JWT + MyBatis。

## 一、项目架构（1 分钟）

项目采用 Maven 多模块结构，共 6 个子模块：

- **techmall-common**：公共模块，封装统一响应格式 Result、JWT 工具类、全局异常处理、分页 DTO
- **techmall-gateway**：Spring Cloud Gateway 网关，端口 8080，负责 JWT 鉴权、角色校验、跨域处理、Sentinel 限流和请求路由
- **techmall-service-user**：用户服务，端口 8081，负责注册登录、用户管理、余额管理
- **techmall-service-product**：商品服务，端口 8082，负责商品 CRUD、分类管理、库存管理
- **techmall-service-order**：订单服务，端口 8083，负责下单、支付、发货、收货，通过 OpenFeign 调用用户和商品服务
- **techmall-web**：Vue3 + Vite + Element Plus 前端

所有服务注册到 Nacos，配置也托管在 Nacos。Gateway 统一入口，JWT 鉴权在网关层完成，下游服务通过 X-User-Id 和 X-User-Role Header 获取用户信息。

## 二、核心技术点展示（2 分钟）

### 1. JWT 鉴权 + Gateway 路由

这是网关的两个核心 Filter：

- **JwtAuthFilter**（GlobalFilter，@Order(-1)）：解析请求头中的 Bearer Token，验证后注入 X-User-Id、X-User-Role 到下游服务
- **RoleAuthGatewayFilterFactory**（自定义 GatewayFilterFactory）：作为 default-filter 应用到全部路由，按路径和方法做角色鉴权，例如 `/api/user/list` 仅 ADMIN、`/api/order` POST 仅 USER

### 2. OpenFeign 远程调用 + Sentinel 熔断

订单服务是整个系统的核心，它需要跨服务调用：

- 调用 **User 服务**：验证用户、扣减余额、增加商家收入
- 调用 **Product 服务**：查询商品价格、原子扣减库存

关键代码在 `OrderServiceImpl.createOrder()`——通过 `@FeignClient` 声明接口，Feign 底层基于 Nacos 服务发现自动负载均衡。同时标注 `@SentinelResource`，配置 fallback 方法，当被调用服务异常时返回"SERVICE_BUSY"提示，实现熔断降级保护。

### 3. Nacos 注册中心 + 配置中心

四个服务启动时自动注册到 Nacos，Nacos 控制台可以看到 `techmall-gateway`、`techmall-user-service`、`techmall-product-service`、`techmall-order-service` 四个实例。Gateway 的路由转发使用 `lb://` 前缀从 Nacos 获取实例列表实现负载均衡。

配置方面，三台业务服务通过 `spring.config.import: optional:nacos:` 从 Nacos 读取配置，支持动态刷新（配合 `@RefreshScope`）。

### 4. MyBatis 动态 SQL + 分页

每个服务的 Mapper XML 使用 MyBatis 动态 SQL 标签 `<where>`、`<if>`、`<set>`，配合 PageHelper 分页插件实现灵活的分页查询。例如商品列表接口支持 `categoryId`、`keyword`、`merchantId`、`includeOffShelf` 多条件组合筛选。

### 5. Sentinel Dashboard 监控

启动 Sentinel Dashboard（端口 8858），可看到各服务的实时 QPS、RT 等指标。网关层配置了 QPS 限流规则，超阈值自动返回 429。

## 三、业务流程演示（1.5 分钟）

### 首页

打开浏览器访问 localhost:5173，展示首页。顶部是 4 页轮播广告，自动切换带渐变动画。下面有 6 大品类的响应式导航网格、8 个真实商品的新品速递区、2 个限时抢购商品。

导航栏搜索 "耳机"——先展示耳机专营店店铺卡片，再列出匹配的耳机商品。

### 用户下单全流程

用 user1 登录（密码 123456），导航栏显示用户标签 + 昵称 + 余额 10,000。

搜索商品 → 进入详情页，可以看到规格标签和商家店铺入口。点击加入购物车 → 填写收货信息 → 提交订单 → 跳转到我的订单列表，状态显示 PENDING。

点击详情进入订单详情页 → 点击"模拟支付" → 余额扣款 → 订单状态变为 PAID → 商家余额增加。

### 商家发货

换到 merchant1 登录，进入订单管理 → 看到刚才的订单 → 点击详情 → "确认发货" → 状态变为 SHIPPED。

### 用户确认收货

换回 user1 → 订单详情 → "确认收货" → 状态变为 COMPLETED。完整闭环结束。

### 管理员功能

用 admin 登录 → 用户管理，可以看到所有用户及其角色。每个 USER 旁边有"订单记录"按钮，可查看和取消该用户的订单；每个 MERCHANT 旁边有"商品管理"按钮，可管理该商家的商品。点击"禁用"商家时，该商家所有商品自动下架。

## 四、数据库设计（30 秒）

三台业务服务各自拥有独立数据库：`techmall_user`、`techmall_product`、`techmall_order`。表结构包括 t_user（含 balance 字段）、t_category、t_product（含 tags 规格标签字段）、t_order、t_order_item（含 merchant_id 冗余字段避免跨库 JOIN）。

数据库间不直接关联，跨服务数据通过 Feign 接口获取，保证微服务的数据自治性。
