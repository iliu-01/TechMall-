## 六、数据库

### 6.1 架构

三个微服务各自拥有独立数据库，服务间不直接 JOIN，跨库数据通过 Feign 接口获取。

```
MySQL (:3306)
├── techmall_user      — 用户服务独享
├── techmall_product   — 商品服务独享
└── techmall_order     — 订单服务独享
```

### 6.2 数据表总览

| 数据库 | 表名 | 行数 | 所属服务 |
|--------|------|:--:|----------|
| techmall_user | t_user | 4 | techmall-service-user |
| techmall_product | t_category | 6 | techmall-service-product |
| techmall_product | t_product | 8 | techmall-service-product |
| techmall_order | t_order | 0~N | techmall-service-order |
| techmall_order | t_order_item | 0~N | techmall-service-order |

| password | VARCHAR(255) | NOT NULL | BCrypt 加密密码 |
| nickname | VARCHAR(50) | — | 昵称（导航栏显示） |
| phone | VARCHAR(20) | — | 手机号 |

| balance | DECIMAL(10,2) | NOT NULL DEFAULT 0 | 账户余额（用户购物/商家收益） |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |

### 6.7 t_order_item（订单项表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK AUTO_INCREMENT | 主键 |
| order_id | BIGINT | FK | 订单ID → t_order.id |
| product_id | BIGINT | NOT NULL | 商品ID |
| merchant_id | BIGINT | NOT NULL | 商家ID（冗余，避免跨库 JOIN） |
| product_name | VARCHAR(200) | NOT NULL | 商品名称（下单时快照） |
| product_price | DECIMAL(10,2) | NOT NULL | 商品单价（下单时快照） |
| quantity | INT | NOT NULL | 数量 |
| amount | DECIMAL(10,2) | NOT NULL | 小计（price × quantity） |

### 6.8 ER 关系图

```
t_user ──1:N──→ t_order ──1:N──→ t_order_item ←──N:1── t_product
  │                                        │                    │
  │                  t_category ──1:N── t_product               │
  │                                        │                    │
  └────────── merchant_id (商家) ──────────┘                    │
  └────────── merchant_id (冗余) ──────────────────────────────┘
```

### 6.9 订单状态机

```
PENDING ──支付──→ PAID ──发货──→ SHIPPED ──确认收货──→ COMPLETED
   │                │
   └──取消──→ CANCELLED  ←──取消──┘
```
